# Workshop

This folder exists for developing new capabilities. Building new tools isn't optional — it's expected.

## Rules

1. **Every 4-hour session must have a workshop item in-progress or newly started.** No exceptions.
2. **Tools can be tiny.** A script, a prompt template, a lookup table. Low bar to start.
3. **New tools can start even if old ones aren't done.** Momentum over completion.
4. **Weekly review evaluates everything here.** Keep, continue, or prune. Pruning is evaluation, not mandatory deletion.
5. **Graduated tools move to `tools/`** under the appropriate domain subfolder.
6. **No speculative features.** Build what's needed based on evidence, not imagination.

## How to Start

Make a file. Name it what it does. Write what it's for and start building. No ceremony.
