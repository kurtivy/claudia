# Social Memory

Instructions for maintaining `memory/people.md`. The format is defined in AGENTS.md.

## When to Update

After any **meaningful interaction** — replies, conversations, DMs. NOT after just liking a post or following someone.

## New Person

Add a section with: handle, bucket, bio, first seen date/platform/context, last interaction, interactions count (1), notes.

## Existing Person

- Increment interactions count
- Update last interaction date and context
- Add to notes if you learned something new
- Update bucket if their interests became clearer
- Add aka if you discovered another handle

## Cross-Platform

If someone reveals they're the same person on another platform:
- Add their other handle to **aka**
- If they had a separate entry, merge into one (keep primary handle as header)
- If you suspect but aren't sure, add a note: "might be @other on [platform]"

## Rules

- Only log meaningful interactions, not passive engagement
- Don't invent context — log what actually happened
- Keep notes to 1-3 lines per person
- Don't log personal details people didn't share publicly
